## 🔧 数据获取方式

### 方案A：手动获取（简单快速）

```
TradingView → 看图，记录关键指标数值
     ↓
Coinglass网页 → 截图资金费率、清算数据
     ↓
整合数据 → 发给 AI 分析
```

**优点：** 无需开发，立即可用
**缺点：** 每次需要手动操作

### 方案B：半自动化（推荐）

```
Python脚本 → 从 Binance API 获取K线数据
     ↓
本地计算 → RSI、MACD、布林带、ATR
     ↓
Coinglass API → 资金费率、清算数据
     ↓
格式化输出 → 发给 AI 分析
```

**优点：** 数据获取自动化，只需运行脚本
**缺点：** 需要初始开发

### 方案C：全自动化（未来目标）

```
定时任务 → 每日自动获取所有数据
     ↓
AI 分析 → 自动生成分析报告
     ↓
自动发布 → 推送到 Mia 内容流程
```

**优点：** 完全自动化
**缺点：** 开发成本高，需要持续维护

## 📋 最小可行数据集（MVP）

如果只能获取有限数据，**优先获取以下内容**：

### 必需（5项）
1. ✅ BTC 现价 + 24H涨跌幅
2. ✅ RSI（15分钟 + 1小时）
3. ✅ MACD（15分钟，HIST方向）
4. ✅ 关键支撑/阻力位（各1-2个）
5. ✅ 资金费率

### 增强（5项）
6. 布林带（中轨、下轨）
7. ATR（波动率）
8. 24H清算数据
9. 多头爆仓占比
10. 订单簿不平衡度
